Workflow Automation n8n
Scrape Data Khách Hàng Doanh Nghiệp

Workflow này thu thập thông tin từ Google Maps, trích xuất email từ website của doanh nghiệp, xử lý qua OpenAI, kiểm tra dữ liệu, và lưu vào Airtable tự động.

Tài Nguyên & Cài Đặt

Tải hoặc copy file JSON cài đặt n8n, hoặc tải file tài liệu hướng dẫn (.docx).

Tải File Hướng Dẫn

Hướng Dẫn Chi Tiết Cài Đặt n8n

1

Manual Trigger (When clicking 'Test workflow')

  • Chức năng: Kích hoạt workflow bằng tay.
  • Cách thêm vào n8n: Nhấn "+" → Chọn Manual Trigger.
  • Kết nối với MAP THU THẬP.
2

MAP THU THẬP (HTTP Request Node)

  • Chức năng: Lấy danh sách các nhà hàng từ Google Maps thông qua API Apify.
  • Cách thiết lập: Thêm node HTTP Request.
  • Method: POST
  • URL:
Bash
https://api.apify.com/v2/acts/compass~google-maps-extractor/run-sync-get-dataset-items?token=YOUR_APIFY_API_KEY
  • Body:
JSON
{
  "categoryFilterWords": ["restaurant"],
  "city": "HA NOI",
  "countryCode": "vn",
  "language": "en",
  "maxCrawledPlacesPerSearch": 10,
  "skipClosedPlaces": false
}
  • Kết nối với Edit Fields.
3

Edit Fields (Set Node)

  • Chức năng: Tạo các trường dữ liệu để sử dụng sau này.
  • Cách thiết lập: Thêm Set Node, chọn Add Field và thêm các trường:
Fields Mapping
Title: ={{ $json.title }}
Street: ={{ $json.street }}
Neighborhood: ={{ $json.neighborhood }}
Website: ={{ $json.website }}
Phone: ={{ $json.phoneUnformatted }}
URL: ={{ $json.url }}
Domain: ={{ $json.website?.extractDomain() }}
  • Kết nối với Loop Over Items.
4

Loop Over Items (Split In Batches Node)

  • Chức năng: Xử lý từng mục một thay vì tất cả cùng lúc (lặp qua từng mục dữ liệu).
  • Cách thiết lập: Thêm node Split In Batches.
  • Batch Size: 1 (để xử lý từng doanh nghiệp một).
  • Kết nối với WEB LỌC.
5

WEB LỌC (HTTP Request Node)

  • Chức năng: Lọc thông tin website và trích xuất nội dung website của doanh nghiệp.
  • Cách thiết lập: Thêm HTTP Request Node.
  • Method: GET
  • URL:
Bash
=https://r.jina.ai/{{ $json.Website }}
  • Kết nối với OpenAI.
6

OpenAI (AI Node)

  • Chức năng: Gửi nội dung website đến OpenAI để trích xuất email.
  • Cách thiết lập: Thêm node OpenAI.
  • Model: GPT-4o-mini
  • Prompt:
Kotlin
Look at this website content and extract only the email I can contact this business.
Provide only the email and nothing else. If no email found, return "Null".
  • Dữ liệu đầu vào:
Bash
{{$node["WEB LỌC"].json.data}}
  • Kết nối với DATAAAA.
7

DATAAAA (Airtable Node)

  • Chức năng: Lưu thông tin doanh nghiệp vào Airtable.
  • Operation: Create Record
  • Base: app8Vn3LTX8ZI1NxW | Table: tblqOs1f4xJU83qSj
  • Fields:
Fields Mapping
Title: ={{ $('Loop Over Items').item.json.Title }}
Neighborhood: ={{ $('Loop Over Items').item.json.Neighborhood }}
Website: ={{ $('Loop Over Items').item.json.Website }}
Phone: ={{ $('Loop Over Items').item.json.Phone }}
Email: ={{ $json.message.content }}
URL: ={{ $('Loop Over Items').item.json.URL }}
Address: ={{ $('Loop Over Items').item.json.Street }}
  • Kết nối với IF Node.
8

IF Node

  • Chức năng: Kiểm tra xem OpenAI có tìm thấy email hay không.
  • Điều kiện: Nếu email chứa "Null", tức là không tìm thấy email, thì tiếp tục tìm bằng API khác.
  • Kết nối hướng 1 (True/Có Email): Kết nối với Airtable1 (để bỏ qua vòng quét phụ).
  • Kết nối hướng 2 (False/Không có Email): Kết nối với MAIL (để tìm email từ nguồn khác).
9

MAIL (HTTP Request Node)

  • Chức năng: Nếu OpenAI không tìm thấy email, thử tìm bằng API anymailfinder.com.
  • Method: POST
  • URL:
Bash
https://api.anymailfinder.com/v5.0/search/company.json
  • Headers: Authorization: Bearer YOUR-API-KEY
  • Body:
JSON
{
  "domain": "{{ $node['Loop Over Items'].json.Domain }}"
}
  • Kết nối với Airtable1.
10

Airtable1 (Cập nhật Record cuối cùng)

  • Chức năng: Cập nhật record đã được lưu ở Airtable nếu tìm được email bằng API anymailfinder.
  • Operation: Update Record
  • Base: app8Vn3LTX8ZI1NxW | Table: tblqOs1f4xJU83qSj
  • Fields:
Fields Mapping
ID: ={{ $('DATAAAA').item.json.id }}
Email: ={{ JSON.stringify($json.results.emails) }}

Chạy thử workflow

  • Nhấn Test Workflow trên Manual Trigger.
  • Kiểm tra đầu ra trên mỗi node để đảm bảo dữ liệu đi đúng hướng.
  • Kiểm tra bảng Airtable để xem thông tin được lưu như thế nào.

Tổng kết - Bạn đã tạo được Workflow tự động:

  • Thu thập doanh nghiệp từ Google Maps một cách nhanh chóng.
  • Trích xuất thông tin website và làm sạch dữ liệu.
  • Dùng trí tuệ nhân tạo (AI/LLM) để tìm email ẩn trong nội dung.
  • Lưu dữ liệu vào hệ thống lưu trữ Airtable.
  • Nếu AI không tìm thấy email, workflow sẽ thông minh dự phòng bằng API thay thế.
